# LMS 과제 발표자료 샘플 (PPT 구성안)

## 1. 프로젝트 개요
- 프로젝트명: **LMS (Learning Management System)**
- 목적: 수강신청부터 학습, 리뷰, 만족도 조사, 사업자(강사) 관리까지 하나의 웹 서비스로 구현
- 범위: 과제용 프로토타입 (핵심 기능 중심 구현)

---

## 2. 사용 기술 (Tech Stack)

### Backend
- **Java 17**
- **Spring Boot 4.x**
- **Spring MVC (Controller + Thymeleaf View)**
- **Spring Security** (로그인/권한 처리)
- **JdbcTemplate + SQL 직접 작성** (핵심 데이터 로직)
- 일부 기능에 **JPA 의존성** 포함 (프로젝트 구성상)

### Frontend
- **Thymeleaf** 서버 사이드 렌더링
- HTML/CSS/Vanilla JS
- 공통 레이아웃(fragment) 기반 UI 구성

### Database
- **MySQL (3308 포트 구성)**
- 기능별 테이블 분리 (courses, enrollments, reviews, schedules, notes, exam_settings, exam_questions 등)

### AI/Chatbot
- OpenClaw Gateway 연동 기반 챗봇 호출
- `/chat`, `/chat/query/stream` SSE 스트리밍 응답
- 강의 컨텐츠 기반 RAG 컨텍스트 주입

---

## 3. 아키텍처/구조 포인트
- 도메인 기준 패키지 분리
  - `course / home / user / security / chatbot`
- Controller-Service-Repository 역할 분리
- SQL 상수 분리 및 기능별 쿼리 명확화
- 페이지 단위 템플릿 구성 + 공통 상단 레이아웃 재사용

---

## 4. 구현 기능 요약

### (1) 사용자 기능
- 회원가입/로그인
- 강의 목록/검색/카테고리 필터
- 강의 상세 조회
- 수강 신청
- 수강 신청 내역 조회
- 학습 페이지 진입
- 학습노트 작성/자동저장
- 수강평 작성/수정/삭제
- 만족도 평가 작성
- 학습 스케줄 관리

### (2) 커뮤니티 기능
- 공지사항 목록/상세/작성/수정/삭제
- 댓글 작성/수정/삭제
- QnA 목록/상세/질문/답변/채택

### (3) 사업자(강사) 기능
- 강의관리 페이지
- 만족도 관리 페이지
- **시험관리 페이지**
  - 시험관리 리스트
  - 시험관리 상세(강의 정보 + 학습목차 + 시험 블럭)
  - 합격점수 변경 (0~100, 5점 단위)
  - 문제 추가/수정 화면

### (4) 시험 기능 (최근 구현)
- 시험 설정 DB: `exam_settings`
  - 강의별 합격점수 저장
  - 기본값 60점
- 시험 문제 DB: `exam_questions`
  - 4지선다 문제 구조
  - 문제/선택지/정답 필수 검증
  - 문제 추가/수정/삭제(UI)

### (5) 챗봇 기능
- 강의 상세/학습 페이지에 챗봇 위젯 임베드
- 사용자별 `sessionTicket` 발급으로 대화 세션 분리
- iframe 기반 위젯 + 컨텍스트 전달(postMessage)
- 스트리밍 응답(SSE)으로 채팅 UX 개선

---

## 5. 적용한 기술적 포인트 (어필용)
- Spring Security 설정으로 페이지 접근 제어
- iframe 정책 대응(`sameOrigin`)으로 위젯 연동 해결
- 기능 단위 테이블 자동 생성(Repository ensureTable 패턴)
- 사용자 경험 중심 UI 개선
  - 초록/화이트 테마 적용
  - 관리 페이지 레이아웃 통일
- 검증 로직
  - 합격점수 5점 단위 제한
  - 시험 문제 필수 항목 검증

---

## 6. 데모 시연 순서 (추천)
1. 메인/로그인
2. 강의 목록 → 강의 상세
3. 수강신청 → 수강신청 내역 → 학습 페이지
4. 챗봇 질의 (강의 컨텍스트 기반)
5. 사업자 로그인
6. 강의관리/시험관리 진입
7. 시험관리 상세 → 합격점수 변경
8. 문제추가/수정 저장 → 문제 수 반영 확인

## 7. PPT에 넣을 캡처 가이드

아래 화면 캡처를 추천합니다.

### 필수 캡처
1. **메인 페이지**
2. **로그인/회원가입 페이지**
3. **강의 목록 페이지**
4. **강의 상세 페이지 + 챗봇 위젯**
5. **수강신청내역/학습 페이지**
6. **사업자 강의관리 페이지**
7. **시험관리 리스트 페이지**
8. **시험관리 상세 페이지 (합격점수 블럭 포함)**
9. **문제추가/수정 페이지**
10. **DB 확인 캡처** (`exam_settings`, `exam_questions` 조회 결과)

### DB 캡처용 SQL 예시
```sql
USE lms;
SELECT * FROM exam_settings;
SELECT * FROM exam_questions ORDER BY course_id, sort_order;
```

## 8. 한계점 / 향후 개선
- OAuth(구글/네이버) 실서비스 키 미연동 상태(과제 범위상 제외 가능)
- 시험 문제 이미지 업로드는 현재 URL 기반 (파일 업로드로 확장 가능)
- 시험 응시/채점 화면은 추가 구현 여지 존재
- 관리자 통계 대시보드 고도화 가능

## 9. 발표 마무리 멘트 샘플
- "본 프로젝트는 학습자와 사업자 양쪽의 핵심 시나리오를 실제로 동작 가능하게 구현한 LMS 프로토타입입니다."
- "특히 시험관리와 챗봇 세션 분리까지 반영해, 단순 UI가 아닌 실제 서비스 구조를 고려했습니다."
- "향후에는 시험 응시/자동채점, 파일 업로드형 문제 자료, OAuth 실서비스 배포 단계까지 확장할 수 있습니다."
