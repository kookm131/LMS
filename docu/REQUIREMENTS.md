# LMS 요구사항 정의서 (완성목표점 기준)

## 1. 문서 목적
이 문서는 현재까지 구현된 LMS 기능을 기준으로,
프로젝트의 **완성 목표점(MVP 완료 상태)** 을 명확히 정의한다.

---

## 2. 프로젝트 범위 (완성 목표점)

### 2.1 포함 범위
- 사용자 인증 (회원가입/로그인/로그아웃)
- 강의 탐색/검색/카테고리 필터/페이징
- 강의 상세 조회
- 수강신청 및 수강신청내역
- 학습하기 화면(목차/시험 진입 조건/성적 계산)
- 커뮤니티 공지사항(목록/고정공지/상세/댓글 CRUD 일부)

### 2.2 제외 범위 (현재 미구현)
- 학습하기 실제 재생/진도 누적 엔진
- 시험 응시 기능(문항 풀이/제출)
- 학습노트 저장/수정
- QnA 게시판
- 마이페이지 상세 기능

---

## 3. 사용자/권한 요구사항

### 3.1 사용자 유형
- 비로그인 사용자
- 로그인 사용자 (학생)
- 관리자(데이터 생성/운영 관점)

### 3.2 접근 제어
- 비로그인 사용자도 강의 목록/상세/공지 조회 가능
- 댓글 작성/수강신청/수강신청내역 조회는 로그인 필요
- 댓글 수정/삭제는 **본인 작성 댓글만 허용**

---

## 4. 기능 요구사항

## 4.1 공통 레이아웃
- 상단 로고/회원 메뉴/메인 메뉴/검색창 제공
- 메인 메뉴:
  - 수강신청
  - 학습하기(hover 드롭다운)
  - 마이페이지
  - 커뮤니티(hover 드롭다운)
- 학습하기 드롭다운:
  - 수강신청내역 (연결)
  - 학습 메모 (미구현)
  - 문의 내용 (미구현)
  - 만족도 평가 (미구현)
- 커뮤니티 드롭다운:
  - 공지사항 (연결)
  - QnA (미구현)

---

## 4.2 메인 페이지
- 추천 강의 슬라이더 제공
- 공지사항 박스 제공
  - 우상단 `+` 버튼 클릭 시 공지사항 목록 이동
  - 고정공지 2개 + 일반 공지 5개 노출
  - 댓글 수와 날짜 노출
- 수강평 박스 제공

---

## 4.3 강의 검색/수강신청 목록
- 내부 검색으로 동작 (외부 포털 이동 금지)
- 검색 조건: 강의 제목 `LIKE %keyword%`
- 카테고리 필터 + 검색어 동시 적용 가능
- 검색 결과 문구 노출:
  - 결과 있음: `xxx에 대한 검색결과입니다`
  - 결과 없음: `xxx에 대한 검색결과가 없습니다.`
- 페이징 제공 (앞 1 2 3 4 5 뒤)

---

## 4.4 강의 상세
- 강의 기본 정보/학습목차/수강평/연관 강의 노출
- 하단 가운데 `목록으로` 버튼 제공
- 우하단 `수강신청` 버튼 제공
  - 클릭 시 confirm: `이 강의를 수강하시겠습니까?`
  - 로그인 시 수강신청 처리
  - 비로그인 시 로그인 페이지 유도

---

## 4.5 수강신청 처리
- 수강신청 시 `enrollments` 테이블 저장
  - `course_id`, `student_id`, 기본 `progress_rate` 저장
- 동일 사용자-동일 강의 중복 신청 방지

---

## 4.6 수강신청내역 (학습하기 > 수강신청내역)
- 로그인 사용자 본인 내역만 조회
- 상단 카테고리 필터 제공 (사이드바 아님)
- 목록 항목:
  - 강의 제목
  - 카테고리/강사
  - 진행률(%) + 신청일시
- 강의 제목 클릭 시 학습하기 상세 페이지 이동
- 페이징 제공 (앞 1 2 3 4 5 뒤)

---

## 4.7 학습하기 상세 페이지
- 상단 제목 블록:
  - 수강 제목/메타
  - 우측 액션 버튼: `학습하기`, `이어서보기`
- 학습목차 블록:
  - 컬럼: 학습하기 / 학습목차 / 학습시간 / 학습상태 / 학습노트
  - 각 목차에 `학습하기`, `이어서보기` 버튼
- 시험 블록:
  - `시험치기` 버튼 제공
  - 진행률 100% 미만 시 경고:
    - `수업 진행율이 100%가 되어야 응시할 수 있습니다.`
- 성적 블록:
  - 진행율 40%
  - 시험 60%
  - 총점 계산 후 표시
- 하단 가운데 `목록으로` 버튼 (수강신청내역 이동)

---

## 4.8 공지사항 목록
- 공지사항 제목/댓글수/날짜 노출
- 댓글수는 숫자만 표시
- 고정공지 2개 항상 상단 고정, bold 처리
- 일반 공지는 최신순 페이징
- 페이징 UI: 앞 1 2 3 4 5 뒤

---

## 4.9 공지사항 상세
- 제목 크게 표시, 본문 표시
- 댓글 목록(페이지당 5개) + 댓글 페이징
- 댓글 쓰기(로그인 시만 활성)
- 하단 가운데 `목록으로` 버튼

### 댓글 권한
- 본인 댓글만 수정/삭제 버튼 노출
- 수정/삭제 시 confirm 창 노출
  - `정말로 수정 하시겠습니까?`
  - `정말로 삭제 하시겠습니까?`
- 수정은 별도 페이지 제공
  - 문구: `수정할메세지`
  - 기존 댓글 내용 미리 입력
  - 우측하단 `수정완료` 버튼

---

## 5. 데이터 요구사항

## 5.1 핵심 테이블
- users
- courses
- enrollments
- lectures
- notices
- notice_comments
- assignments, submissions
- quizzes, quiz_results
- course_reviews

## 5.2 정렬 원칙
- 공지 목록/메인 공지: 최신순(기본 id DESC)
- 댓글 목록: 최신순

## 5.3 데이터 보존
- 서버 재기동 시 데이터 보존되어야 함
- 초기화 SQL 자동실행 비활성화(`spring.sql.init.mode=never`)

---

## 6. UI/UX 요구사항
- 목록 버튼은 화면 하단 가운데 정렬 통일
- 버튼/블록 스타일은 페이지 간 일관성 유지
- hover 드롭다운은 상단 메뉴와 정렬 일치

---

## 7. 비기능 요구사항
- 입력값 검증(공백 댓글 방지)
- 권한 검증(본인 댓글만 수정/삭제)
- 페이지 범위 보정(잘못된 page 파라미터 방어)
- 에러 시 안전 리다이렉트(존재하지 않는 상세 접근 방지)

---

## 8. 완료 기준 (Done)
아래 조건을 모두 만족하면 본 문서 기준 완성목표점 달성으로 본다.

1. 로그인 후 강의 상세에서 수강신청 가능
2. 수강신청내역에서 본인 데이터/카테고리 필터/페이징 정상 동작
3. 학습하기 상세에서 목차/시험 진입조건/성적 계산 표시
4. 공지사항 목록/상세/댓글 작성/수정/삭제 권한 동작
5. 메인 공지 박스(고정+일반) 및 이동 동선 정상
6. 서버 재기동 후 작성 데이터 유지
